# Security Profile

I build software with a cybersecurity perspective.

My background combines machine learning, backend engineering, and product delivery, but the thread that matters most is this: systems should remain understandable, observable, and trustworthy when conditions get messy.

## What I care about

- Threat-aware engineering for APIs, data pipelines, and ML-backed products
- Explainable and production-safe machine learning workflows
- Secure development habits that reduce silent failure and long-tail risk
- Practical defensive thinking instead of security theatre

## Current direction

I am especially interested in roles and projects involving:

- cybersecurity engineering
- secure AI systems
- model integrity and monitoring
- backend reliability and hardening
- DevSecOps and security-minded delivery workflows

## Working style

I prefer clear systems, measurable outcomes, and designs that are resilient under operational pressure. That means asking where trust boundaries sit, how failure shows up, what should be logged, and how a feature could be abused before it is shipped.
